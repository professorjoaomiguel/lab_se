# 🤖 Guia de Uso Saudável de IA (Mentor, não Autor)

Nesta disciplina, incentivamos o uso de Inteligência Artificial (Gemini, ChatGPT, etc.) como um **Tutor Pessoal 24h**. No entanto, o objetivo é que você desenvolva a "musculatura cerebral" necessária para um engenheiro.

### ✅ O que é uso SAUDÁVEL (Aprender):
1. **Explicação de Conceitos:** "Explique como funciona o ciclo de trabalho (Duty Cycle) no PWM do MicroPython."
2. **Depuração (Debug):** "Meu código está dando `MemoryError`. Quais são as causas comuns no ESP32?"
3. **Pistas, não Soluções:** "Estou tentando fazer o Experimento 03. Não me dê o código, mas me dê uma pista de como implementar a Histerese."
4. **Leitura de Datasheet:** "O LM35 diz que entrega 10mV/°C. Como eu converto isso para uma fórmula matemática?"

### ❌ O que é uso PREJUDICIAL (Copiar):
1. **Prompt Direto:** "Resolva o experimento 04 para mim e me dê o código pronto."
2. **Falta de Crítica:** Copiar código sem entender o que cada linha faz.
3. **Alucinação:** Seguir orientações da IA para bibliotecas que não existem no MicroPython (ex: usar bibliotecas de Arduino C++ no Python).

---

### 💡 O "Prompt Mestre" (Use isso!)
Para que a IA te ajude sem tirar seu aprendizado, comece sua conversa com ela assim:

> "Atue como um Professor de Sistemas Embarcados. Estou aprendendo MicroPython com ESP32. Vou te enviar o objetivo de um experimento. **Não me dê o código pronto.** Em vez disso, explique os conceitos envolvidos e me dê pequenos desafios ou pistas para que eu mesmo chegue à solução."

---

### 🚩 O Teste do "Por Quê?"
Em qualquer momento, o professor pode pedir para você explicar uma linha específica do seu código. Se você não souber explicar o porquê daquela linha estar lá, o exercício será considerado inválido.
